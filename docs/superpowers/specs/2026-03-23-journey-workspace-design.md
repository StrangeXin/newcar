# 旅程工作台设计 spec

**日期**: 2026-03-23
**状态**: 已批准
**作者**: 产品 + Claude Code

---

## 1. 概述

本文档定义旅程工作台（Journey Workspace）的视觉和交互设计。工作台是用户在购车旅程中的核心操作界面，包含 AI 聊天面板、旅程进度、候选车管理和信息摘要四个模块。

---

## 2. 整体布局

### 2.1 三栏布局（≥1280px）

```
┌─────────────┬──────────────────────────┬───────────────┐
│  左侧边栏    │       中间看板            │   右侧聊天    │
│  220px      │       flex-1             │   340px      │
│  旅程进度    │   今日动态/AI摘要/候选车   │   AI 助手    │
└─────────────┴──────────────────────────┴───────────────┘
```

### 2.2 响应式断点策略

| 视口 | 布局 | 说明 |
|------|------|------|
| **≥1920px** | 260px + flex + 400px | 侧栏放宽，看板区更宽，字号/间距同步放大 |
| **1280–1919px** | 220px + flex + 340px | 基准三栏布局 |
| **1024–1279px** | 180px + flex + 280px | 侧栏压缩，AI置信度/发布按钮/快捷芯片隐藏 |
| **768–1023px** | 1fr + 300px | 两栏，左侧进度条改为横向一行展示，聊天保持 |
| **<768px** | 单栏 | 左侧隐藏，聊天在移动端通过 FAB 触发 |

## 3. 左侧边栏：旅程进度

**组件**: `StageProgress`

- 垂直阶段列表，当前阶段高亮（黑色背景+白色文字），已完成（绿色+勾号），未开始（灰色）
- AI 置信度仪表：紫色渐变进度条 + 百分比数值 + 简短描述（1024px以下隐藏）
- 发布按钮：白底细边框，1280px以上显示

### 阶段顺序

1. AWARENESS（需求确认）
2. CONSIDERATION（候选筛选）
3. COMPARISON（深度对比）
4. DECISION（决策强化）
5. PURCHASE（购买执行）

---

## 4. 中间看板：信息与候选车

**组件**: `Kanban` → `TodayUpdates` + `AiSummary` + `CandidateCards`

### 4.1 今日新动态（TodayUpdates）

- 图标 + 标题 + 简述 + 时间戳
- 状态：未读高亮（白底灰边），已读淡化
- 点击标记已读

### 4.2 AI 旅程摘要（AiSummary）

- 叙述性摘要文字（narrativeSummary）
- 关键洞察列表：按置信度分级颜色（高=绿色，中=蓝色，低=黄色），显示 insight + evidence + 置信度%
- 下一步建议：可点击操作芯片（action chips）

### 4.3 候选车型卡片

每张卡片包含：
- 车型图标（emoji + 渐变背景色）
- 车型名称 + 类型 + 价格
- AI 匹配度进度条 + 百分比
- AI 推荐备注（紫色背景）
- 操作按钮：选定 / 淘汰 / 备注

**淘汰态**：整卡 opacity 50%，名称删除线，备注显示淘汰原因，操作按钮变为"恢复候选"。

## 5. 右侧聊天面板

**组件**: `ChatPanel` → `MessageList` + `MessageBubble` + `ChatInput`

### 5.1 消息样式

**用户消息**：右对齐，深色气泡（#1a1a1a），圆角 14px 14px 2px 14px

**AI 消息**：
- 文字气泡：米白背景（#f5f4f2），圆角 2px 14px 14px 14px
- 富卡片：车型卡片内嵌在消息流中（详见 5.2）
- 流式输出：末尾显示紫色闪烁光标 `|`

### 5.2 消息类型

**文本消息**：纯文字回复

**富卡片消息**（AI 推荐候选车时）：
```
┌─────────────────────────────────┐
│ [🚗] 理想 L6                    │
│       增程 · 五座 · 27.98万起    │
│       ████████░░ 92%           │
│                        [+ 加入] │
└─────────────────────────────────┘
```

### 5.3 工具调用状态

当 AI 正在调用工具时，在消息下方显示内联状态条：

```
🔍 正在搜索「30万 增程 SUV」… ●●○ (跳动点)
```

- 紫色背景（#faf5ff）+ 紫色边框
- 工具完成后自动淡出（200ms transition）

### 5.4 副作用确认芯片

AI 执行有副作用的操作（如加入候选车）后，显示确认芯片：

```
✓ 理想 L6 已加入候选列表    [查看]
```

- 绿色背景（#f0fdf4）+ 绿色边框
- "查看"链接可点击跳转

### 5.5 聊天输入区（升级版 A）

**结构**：
```
┌──────────────────────────────────────────┐
│ [📊 帮我对比] [💰 预算调整] [🔍 再搜一批] │  ← 快捷芯片（可点击）
│ ┌────────────────────────────┐  ┌────┐  │
│ │ 问我任何购车问题…          │  │ ↑  │  │  ← 输入框 + 内嵌发送
│ └────────────────────────────┘  └────┘  │
└──────────────────────────────────────────┘
```

- 快捷芯片：灰色胶囊按钮，首次打开时显示，引导用户常用操作
- 输入框：单行最小高度，Enter 发送，Shift+Enter 换行
- 发送按钮：深色圆形/圆角，位于输入框右侧

## 6. 设计规范

### 6.1 色彩

| 用途 | 色值 |
|------|------|
| 背景 | #f0ede8（主背景）/ rgba(255,255,255,0.88)（面板） |
| 文字 | #111（标题）/ #333（正文）/ #6b7280（次要） |
| 主色 | #6366f1（AI匹配度高）/ #8b5cf6（AI主题） |
| 成功 | #22c55e（完成）/ #16a34a（文字） |
| 警告 | #eab308（洞察低置信度） |
| 工具态 | #7c3aed（工具调用中）/ #faf5ff（背景） |
| 副作用 | #15803d（已加入）/ #f0fdf4（背景） |

### 6.2 圆角

- 面板/卡片：16–20px
- 消息气泡：14–16px
- 按钮/芯片：7–10px / 20px（胶囊）

### 6.3 阴影

- 主卡片：`0 8px 30px rgba(18, 18, 18, 0.08)`（shadow-card）
- 次级卡片：`0 1px 6px rgba(0,0,0,0.07)`
